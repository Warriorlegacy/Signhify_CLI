# Signhify Acceptable Use Policy

Signhify is an open-source AI coding agent. While we believe in freedom to use software for any lawful purpose, we prohibit certain uses to prevent harm.

## Prohibited Uses

You may not use Signhify for:

1. **Illegal activity** — any violation of applicable law or regulation
2. **Malicious code** — generating malware, ransomware, viruses, or other harmful software
3. **Unauthorized access** — hacking, credential theft, or bypassing security controls
4. **Military applications** — weapons systems, drone control, or combat-related software
5. **Infringement** — violating intellectual property, privacy, or contractual rights
6. **Harassment/discrimination** — generating content that harasses, threatens, or discriminates
7. **Autonomous high-risk decisions** — operating critical infrastructure, medical devices, or financial systems without human oversight
8. **Deception** — impersonation, fraud, or generating disinformation
9. **Child safety violations** — any content involving the exploitation of minors
10. **Personal data processing** — unauthorized collection or processing of personal data

## Responsible Use

- Review code generated by Signhify before running it, especially when it has file-system or shell access
- Configure shell allowlists/denylists appropriate to your security needs
- Use `--auto` mode only in CI/CD or other sandboxed environments
- Keep your API keys secure — never commit them to version control

## Reporting Violations

If you discover a use case that violates this policy, report it to **security@signhify.dev**.

## Enforcement

Violation of this policy may result in blocked access to Signhify-related services (npm packages, VS Code extensions, community forums) and, where applicable, referral to law enforcement.
